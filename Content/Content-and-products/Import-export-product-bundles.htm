Importing and exporting product bundles

You can import and export product bundles by an Excel spreadsheet. You can either export all current product bundles, or you can export a spreadsheet to use as a template when importing product bundles.

When updating by spreadsheet, you should always export first to ensure you are using the most recent spreadsheet version. This will also guide you on the proper format for each column and row as well as which fields are required.

Be sure your browser is set up to allow pop-ups from the site manager.
To export product bundles
  1. In Content, select Import/Export > Product Bundles.
  2. Choose to export either current bundles, or a blank template:
    1. To export a spreadsheet with your current product bundles, click the Export Current Bundles to Excel button. This generates an Excel spreadsheet, and a link to it at the bottom of the page. Click this link to open, modify, and or save it to your hard drive. See Product bundles spreadsheet fields for field descriptions.
    2. To export a spreadsheet you can use as a template, click the Export Empty Template to Excel button. This generates a template with sample records for the first 10,000 products in to allow for a quick setup of product bundles.

To import product bundles
  1. In Content, select Import/Export > Product Bundles.
  2. Click Browse, find the Excel spreadsheet you want to import, and click Open.

  3. Click the Import button .

    Product bundles are automatically uploaded to the server, and each new product bundle is assigned a bundle ID used to identify it in the database.

 Import notes
  • Each product_id can be listed more than once—once per bundled item in the bundle.
  • If product_id is invalid, then ignores it, and displays an error message at the end of processing.
  • If Bundle_Id is invalid, then ignores it, and displays an error message at the end of processing.

  • If a specific row fails to import, it is skipped and an error message describes the failure reason.

  • During the import, read-only fields will be ignored, as they are not required.
