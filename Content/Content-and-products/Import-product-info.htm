Importing product information

This feature allows you to bulk import products without options, or edit existing ones, using the Aptos pre-formatted spreadsheet.

Product and pricing spreadsheets are processed by Aptos . They can be scheduled for processing in advance by up to 90 days. All columns are defined in Exporting product information.

When updating by spreadsheet, you should always export first to ensure you are using the most recent spreadsheet version. This will also guide you on the proper format for each column and row as well as which fields are required.

About pending tasks and history

The screen contains two areas of information.

  • Pending Product Imports spreadsheet(s).
  • The Product Import History (past 90 days) worth of imports that have been processed.

If a pending import has not been processed or is not currently in process, you can cancel it here.

When performing a product import and you analyze the data first, it is important to know that the amount of products in the spreadsheet cannot exceed 5,000 items.

To cancel an import

Click the Delete button corresponding to the import file you want to cancel. You will be asked to confirm your action and the task will be canceled, if desired.

To import product information

  1. In Content, select Import/Export > Import Product.
  2. Click Browse, find the Excel spreadsheet you want to import, and click Open.

  3. If you are importing an inventory only spreadsheet, then click on the Inventory Update Only button, and go to step 7.

    This type of spreadsheet can be generated on the Product Export menu .

  4. To review the products on the screen before importing them, click Analyze Import. Remember, there is a 5,000 item limit when importing from a spreadsheet.

    If you are an expert Aptos spreadsheet user, you may opt to Import without Analysis but this is not recommended.

  5. When analyzing the imported spreadsheet, if finds that the ImagePath field for any record contains "http://" , it prompts with the question:

    How should we handle ImagePath values that contain a URL?

    1. Download images from the remote server and store them on the Aptos server
    2. Always display the image hosted at the remote location

    By default, the Download images... radio button is selected.

    If you select Always display..., stores the URL as the ImagePath value. The standard pages will be updated to include logic to display the product image from a remote server as needed.

  6. When the analysis is done, you are presented with the results. After your products are displayed on the screen, you may manually adjust the sub-category to which the product belongs, correcting any misspellings. You can also see if your images' paths are correct.

  7. Click the Import button after all products load.

    The larger the spreadsheet, the longer it will take for this screen to load.

  8. Make any changes needed to the export schedule date and time, and notification email address if needed, and click Save.

    ​

    The file is uploaded and renamed with a unique name, is scheduled for processing, and appears as a pending import in the Pending Product Imports area of the main screen.

  9. When the job is complete, the update appears in the Product Import History area of the main screen. ​Click the Details link in the Status column to view the Job Status details.

    The job history page indicates the import status from the email subject: OK, OK (WITH ERRORS), or ERROR. The data logs the row count successes and failures along with detailed logging information.

Sub-Category auto-generation

  • When a conflict exists where the sub-category in the spreadsheet does not exist in the database, the import (just like brands currently works), will creates the sub-category on the fly and the product auto-associated to it. The wording on the screen in the drop down box makes this more evident by saying Create Sub-Category.

  • If the sub-category contains a /, it will be replaced with a hyphen during import.

Product UPC

You can use Product With Options Import and Export to define both the product-level and option-level UPC values in the spreadsheet.

  • ProductUPCCode - This new column in the spreadsheet, correlates to the product-level UPC Code field.

  • UPCCode - This existing column correlates to the option-level UPC field.
