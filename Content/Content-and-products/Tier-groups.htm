Tier groups

Use tier groups to provide credits based on the quantity purchased and assign fees based on the dollar amount spent.

Searching for existing tier groups

In Content > Customers > Tier Groups, you can search for existing tier groups by tier group name or short description. You can further filter the search by selecting the tier status from the provided drop-down menu. Select the number of tier groups you want to be displayed per page. Next, click the Search button. A list of current tier groups will be displayed based on your search parameters.

 Adding a tier group
  1. To add a new tier group, in Content > Customers > Tier Groups, click the + Add button. A new window opens.

  2. Enter Group Configuration information:

    Group Configuration settings
    Item Description
    Group Name (Required) A name for the tier group that will easily identify what type of members will belong to the tier group or what function the group will serve. For example, Employees.
    Short Description A short description for the tier group.
    Members Area Header This header will appear at the top of the members area as a way letting customers know what tier group they belong to.
    Members Area Description Appears in the members area to provide customers with a brief description of the tier group they are in.
    Show in Members Area If selected, tier group information will appear in the members area.
    Optional Image Upload an image for the tier group. This will display depending on wiring.
    Custom Description If wired to support a custom description, enter it in this field.
    All orders must be held to ship complete If selected, orders in this tier group must have all items available for shipment before the order can be fulfilled.
    Tier Status You can select the tier status as either valid or invalid. If the tier group is set to invalid, tier properties will not be applied to members assigned to that tier group.

  3. Click the Payment Types Show Details... link, and enter payment details. This section allows you to restrict members of a tier group to a specific payment type. Select the payment types you want to limit members of the tier group to. If you make no selections, customers may use any payment type.

  4. Click the Billing/Shipping Addresses Show Details... link, and add billing and shipping addresses that can be used on an order. If no addresses are added, then a customer in the tier group can use any billing and shipping address associated with their account. If you want to restrict tier group members only to specific shipping addresses and billing addresses, click the Add button. You can additionally choose to restrict tier group members to specific shipping addresses only, or to specific billing addresses only, by selecting the related check boxes.

  5. Click the Pricing Rules Show Details... link, and enter pricing rules:

    Pricing rules settings
    Item Description

    Tier Discount

    This discount applies to all products assigned to the tier.

    Non-Tier Discount

    This discount applies to all products not assigned to the tier.

    Subscription Discount

    This discount applies to all products that are on a recurring subscription program.

    Maximum Credit Card Allowed

    When this amount is set, during the checkout process, credit card is disallowed for users within this tier group if the amount of the cart exceeds this dollar amount.

    Credit Card Fee or Discount

    If the credit card payment type is used at checkout, the configured fee ( + ) or discount ( - ) will be applied to the order. This can be applied as either a percentage amount or a dollar amount.

    Cash Payment Fee or Discount

    If the cash payment type is used at checkout, the configured fee (+) or discount (-) will be applied to the order. this can be applied as either a percentage amount or a dollar amount.

    Quantity Discount

    This discount applies to Cart if the quantity of all items in Cart reaches or exceeds set quantity. This can be applied as either a percentage amount or a dollar amount.

    Apply if Cart Quantity Reaches

    Set the threshold value for the Quantity Discount in this field.

    Cart Fee

    This fee will be applied to the Cart if the Cart falls below set value.

    Apply if Cart Falls below

    Set the threshold value for the Cart Fee in this field. The value can be Subtotal or Quantity.

    Shipping and Handling Fee

    When this amount is set, during the checkout process, this fee will be added to all orders.

    Maximum Order Amount Allowed

    If configured, customer is not allowed to place an order if current cart subtotal plus cart subtotals of previous orders within set rolling period exceeds the Maximum Order Amount.

    Apply Within Rolling Period Of

    Set the rolling period for the Maximum Order Amount Allowed in this field. This can in days, hours, or weeks.

    Percentage discount applies to 'On Sale' price

    Select to add the tier group percentage discount on-top of the On Sale price when a product is on sale.

    Free Shipping on all orders

    Select to apply free shipping to all orders placed by tier group members.

    Minimum Order Value

    If configured, customer will not receive Free Shipping if the cart subtotal is less than the Minimum Order Value amount

    Tax Exempt

    Select to make to tax exempt all orders placed by tier group members.

    Allow Backorders

    Select to allow order items to be backordered. Items not on backorder will be processed normally.

    Hold all orders

    Select to hold all orders placed by tier group members.

  6. Click the Associated Affiliates Show Details... link, and assign affiliates to the tier group by selecting them from the list. Assign multiple affiliates by holding down the Ctrl key when highlighting. Note that this list will populate only with the affiliates that have been setup in Marketing > Affiliates > Add/Edit.

    Select the Affiliate(s) requires the consumer to registered/cookied in order to view products in this tier. check box to require customers to be logged in to view affiliate products. If the customer is not logged in, or does not have a cookie for "Customer ID" they will be redirected to the proper URL to sign in. Once the customer signs in, they will be redirected back to the specific product page they were previously on. The cookie for "Customer ID" is populated upon login then stored, and can last for up to 180 days, thus allowing the system to know that they have signed in once before.

    1. If Redirect to: field is populated - The customer will be redirected to the URL provided. So, if you enter customlogin.asp in the Site Manager, the customer will be redirected to https://SiteUrl/customlogin.asp.

    2. If Redirect to: field is NOT populated - The customer will be redirected to https://SiteUrl/memberlogin.aspx.

  7. Click the + Add button to create the tier group. The new group appears in the list with any others already created.
 Editing current tier groups

A list of current tier groups will be displayed depending on your search criteria. Here, you can edit and manage different aspects of each tier group. Note that you can delete a tier group at any time by clicking the Delete button. To Edit tier group rules, admins, members, and taxonomy, click on the related pencil icon . A popup window will appear.

To edit rules

In the Rules window, you can edit any of the setting configured when the tier group was first created. In addition, you can assign trigger products to the tier group.

  • At the bottom of the window, click Trigger Products to expand the panel.
  • If a customer places a trigger product in their shopping cart, then that customer will automatically become enrolled in the associated tier group and receive any configured discounts or restrictions. To assign trigger products to a tier group, click the Add button. Locate the desired product, and enter the number of days that a product will serve as a trigger product for that tier group.
  • You can delete a trigger product from the tier group at any time by clicking the Delete button. When you are finished adding trigger products, click the Save button.
 To edit administrators

Use this feature to assign customers as Tier Group administrators. You can also define certain permissions for these administrators. You can give these administrators the capability to see the order history for orders that have been assigned to their Tier Group.

A single customer can be the administrator to multiple Tier Groups. However, they can only be assigned to one Tier Group at a time.This functionality is especially useful for B2Bbusiness-to-business sites, as there can be multiple user accounts associated with a parent-level account, or Tier Group.

  1. To assign Tier Group administrators, click the pencil icon under the Admin column for the tier group you want to assign.The Tier Group Admins window opens.
  2. Search for the customer to Add Customers to Current Tier Group as an Admin and if needed, to later define permissions.

  3. Select the customers you want to add and click (+) Add selected Customer(s) to current tier.

  4. Close the window. You will be returned to the previous page. The names you selected will display in the Current Admins area.
  5. To manage permissions for an administrator, in the Permissions column, click the Manage Permissions (pencil icon). The Manage Permissions modal window opens.
  6. Select the items you want the administrator to control for this Tier Group. Save your changes.

You can also Import Administrators From Excel to Current Tier Group if you want to set up many customers as administrators. Be sure to complete the required fields in the spreadsheet. As usual, it is a good idea to export the current Excel spreadsheet, complete it, and then import it back into the site manager.

Currently, you can only import members as administrators for the current tier group selected.

To edit members
  1. Click the pencil icon under the members column for the Tier Group you want to manage members. The Tier Group Members window opens.
  2. Search for the customer to Add Customer(s) to Current Tier Group and if needed, to later add shipping addresses.
  3. Select the customers you want to add and click on (+) Add selected Customer(s) to current tier. Close the window. You are returned to the previous page. The names you selected appear in the Current Members area.

  4. You will be returned to the previous page. The names you selected will display in the Current Members area.

  5. To manage shipping addresses for a member, in the Tier Shipping Addresses column, click the Manage Tier Shipping Addresses (pencil icon). The Manage Tier Shipping Addresses window opens.

  6. Select the items you want the customer to be associated with for this tier group. Save your changes.

You can also Import Customers From Excel to Current Tier Group if you want to set up many customers at a time. Be sure to complete the required fields in the spreadsheet. As usual, it is a good idea to export the current Excel spreadsheet, complete it, and then import it back into the site manager. Select the check box to send a welcome email to the new members.

If the TierGroupID field is left blank, the customer will be imported as a member of the currently selected tier group only. However, you can assign a customer to any tier group by entering in the TierGroupID number. If a customer is already a member of another tier group, entering a different TierGroupID number will remove them from the current tier and assign them to the new TierGroupID. Remember that a customer may only be a member of one tier group at a time.

To edit taxonomy

Click the Taxonomy icon to manage Tier Group taxonomy. Once clicked, a modal window will open from which you can select to manually adjust the taxonomy for the given tier group. This is useful when you want someone belonging to a tier group, or an affiliate tied to a tier group, to see a specific taxonomy.

There are three specific taxonomy groups that can be modified. You don't need to click the "Select All" link or individually check all the Categories, Sub-Categories, brands, and products in order to assign them to a tier. By default, no checkmarks serve the same function as if all checkboxes are marked.

  • Categories/Sub-Categories - You can filter the taxonomy group to show specifically which category structure you want the customer to see. By default if none are specified, they will see all Categories/Sub-Categories.
  • Brands - You can filter to only show those brands that are allowed to be seen.
  • Products - You can also get very granular down to the product level to filter the allowable products for the customer. These settings will also affect the customer's search results from displaying these products.
 Importing and exporting tier groups
You should always do an export to ensure you are using the most recent spreadsheet version. This will also guide you on the proper format for each column and row as well as which fields are required.
To export tier groups

To export a spreadsheet with your current tier groups, click the Export to Excel button . A green success message appears at the top of the page, containing a link to the Excel file. Click the link to either save the spreadsheet to local storage, or open it for editing. Be sure your browser is setup to allow pop-ups from the site manager.

 To import tier groups
  1. Click the Choose File button to select the local source where your import excel spreadsheet resides.
  2. Click the Import button . The tier groups will be automatically uploaded to the server. Each new tier group will be assigned a TierGroupID that will be used to identify it in the database.
