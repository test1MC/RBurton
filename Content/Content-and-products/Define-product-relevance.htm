Defining product relevance

Use product relevance to define the order in which your products will appear within each Sub-Category. The product relevance functionality allows you to easily define this order, and to update product names if needed. If no display order is defined, your products are displayed alphabetically.

Note: Your site must be wired to accommodate sub-category display order. If you wish to define this order, contact your Client Manager to discuss further.

There are two ways to define product relevance:

  • using the Standard View, where you drag and drop products to define order of relevance,

  • using the Extended View, where you manually rank relevance order by number. Extended view displays more product information, which you can edit as well.

To define product relevance in Standard View
  1. In Content, select Sub-Categories > Product Relevance.
  2. Select a Sub-Category from the list for which you would like to define the order of the products.

  3. Choose whether to Exclude Products with "0" quantity.

    Note: This check box is only associated with the value (inventory) for the product in Content > Products > Add/Edit for a specific product's Quantity field. If the product has a supplier or options that have inventory, this field doesn't apply.

  4. Select Standard View, and click Search.

  5. Drag products to reflect the order of relevance you want.

  6. To open a modal window in which you can find and select products to share into the selected sub-category, click the Share Product into Current Sub-Category (+) button .
  7. Click Save when done.
 To define product relevance in Extended View
  1. In Content, select Sub-Categories > Product Relevance.
  2. Select a Sub-Category from the list for which you would like to define the order of the products.

  3. Choose whether to Exclude Products with "0" quantity.

    Note: This check box is only associated with the value (inventory) for the product in Content > Products > Add/Edit for a specific product's Quantity field. If the product has a supplier or options that have inventory, this field doesn't apply.

  4. Select Extended View, and click Search.

  5. Make changes as needed:

    1. Display Name - edit the name of the product if desired.     2. Brand - brand name of the product.

    3. Active - indicates whether a product is active or inactive.

    4. Sells For - the price the product sells for on your site.

    5. On Sale - the on sale price if relevant.

    6. Type -if the product is assigned to the selected sub-category, no value is displayed in this column. If the product is assigned to a different sub-category and shared into the selected sub-category, shared text is displayed in this column.     7. Order - enter the order in which you would like the product to display on the Sub-Category page.

    8. Share Product into Current Sub-Category - Click the (+) button to open a modal window in which you can find and select products to share into the selected Sub-Category.

  6. Click Save when done.
 About changing product names and 301 redirects

When a product name changes, 301 redirectsA method of telling web browsers and search engines that a web page or site has been permanently moved to a new location. Usually a 301 redirect includes the address to which the resource has been moved. need to be created to ensure that SEOsearch engine optimization is not lost. By creating 301 redirects, ensures that when you change the display name of a product, any potential customers who have bookmarked "liked" or stored the page will automatically be transferred to the new product page. This reduces 404 errorsPage Not Found errors that could lead to lost sales.

After changing and saving product display name changes, the Create 301 Redirect pop-up window opens. It notes that the system has created a new page with a new URL for each changed product. Select the products for which you want to create 301 redirects. After making all the relevant changes, click Save located at the button of the page.
