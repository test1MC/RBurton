Importing and exporting QuickSheet data

Use the QuickSheet to quickly update the specified products with options fields available based on a given key field. With the QuickSheet you can specify a key field along with other fields to update. The QuickSheet focuses especially on inventory and pricing, but it can be used for several functions. This is a great way to quickly update fields in the platform via a spreadsheet without the need to re-import the entire product or product with options spreadsheet. To learn more about the products product with options spreadsheet, the document Using the Products with Options Spreadsheet. This guide defines all the columns in the spreadsheet, identifies required fields and explains the information needed for each field.

Use QuickSheet to update existing products but not to import new products.

About the Select Key

updates any item at both product and option level that matches the key field. There are three available keys that can be used. These are:

  • Product/Option SKU
  • Alternate SKU
  • Product/Option UPC Code

To import/export data using QuickSheet

  1. In Content, select Import/Export > QuickSheet.
  2. Select the key to use from the list.
  3. Select the Fields to Update; hold down the Ctrl key to select more than one.

  4. In the Spreadsheet Row Size field, enter the number of records you plan to import and/or update.

    This number is necessary only if you are planning on manual data entry, with the default of 500 rows.

  5. Click the Download Spreadsheet Template button .

    The spreadsheet opens in the bottom of the window, and the Import, Export and Paste functions become available.

  6. Use the appropriate update method; it is similar in concept to that of Google Docs:

    1. Paste (with Internet Explorer)link – Use Paste from Excel to copy and paste rows and columns directly from an Excel spreadsheet into the spreadsheet in the Site Manager. The paste functionality only works with Internet Explorer To do this, click in the starting cell and click the Paste from Excel (IE Only) link, and the page reloads the spreadsheet with the pasted content.

    2. Export templatebutton – When you have determined the fields you want to update, you can export the template you created, complete with the column headings. When saved successfully, you will see a message that the QuickSheet has been created successfully. You can use template can be used to make edits and import it back into the system.

    3. Import button – When your spreadsheet is set up on the screen with both the key and selected fields you want to update, you can import an Excel spreadsheet into . Click the Browse button, find the file, and import it. The file must be in .xls format and the first sheet must be named Sheet1. After importing the file, it will re-load the grid with the saved content.

    4. Manual data entry – You can quickly enter the in the Key and Update Fields and allow you to update the database accordingly. If you make manual entries, note that Google Chrome and Firefox do not allow for single cell updates without posting data back and re-loading.

  7. Click Save when done.
QuickSheet spreadsheet fields
Field Description
Active Exists at both the product and product option level. When added to the spreadsheet, just as with the other existing fields such as quantity checks against the key field at both product and option level to update. Valid values for this field include 0/1 or true/false.
Deleted Exists at only the product level. Valid values for this field include 0/1 or true/false.
Bin Number Exists at product and product option level.
Classificiations/ Attributes

Exists at product and product option level. For Classification/Attributes, match the format used in spreadsheet importing

Color=Black,Grey|Features=Impact Resistant,Water Resistant
Cost Exists at product and product option level. Valid values for these fields are numerical.
Display Order Used to view, add, or change the display order of products and/or products with options.
Expiration Date Exists at product level. Valid values for these fields are numerical.
Long Description Product details or features. Exists at product level. Valid values for these fields is text.
MAP Price Exists at product level. Valid values for these fields are numerical.
Meta Description Exists at product level. Valid values for these fields is text.
Meta Keywords Exists at product level. Valid values for these fields is text.
Meta Title Exists at product level. Valid values for these fields is text.
MSRP Exists at product level. Valid values for these fields are numerical.
MSRP Max Exists at product level. Valid values for these fields are numerical.
On Sale Price Exists at product and product option level. Valid values for these fields are numerical.
Selling Price Exists at product and product option level. Valid values for these fields are numerical.
Shipping Add-On Amount Exists at only the product level. Valid values for this field include any decimal amount up to 2 places (for example, 99.50).
Quantity Exists at product and product option level. Valid values for these fields are numerical.
Tier Group ID Used to update Tier Pricing for products. When you put a product in a Tier Group, the tier price is required.
Tier Price Exists at product level. Valid values for these fields are numerical.
Extra 1 - Extra 10 Exists at product level and product option level depending upon how they are used.
Display Name Used to edit or change the front end display name of products. In addition, a 301 redirect will be automatically created.
Editors Choice When true, and your site is wired accordingly, the product shows as an editor's choice or featured product.
Image Name The name of the image file you would like to display. Enter a file path if necessary.
BundleItemsShipTogether With this rule in place, if any item in the bundle is on backorder, all items in the bundle will be placed in backordered status until all items are available to ship. This rule applies to both product bundles and cross-sell groups set to 'Must Ship Together'. A true value sets the 'Must Ship Together' flag. A false value removes the flag. By default, or if column is blank or NULL, column will be set to false.
ProductIsDonation

You can take donations from your customers for various causes at checkout. This is a digital good which appears on the Place Order tab of checkout. In addition to configuring charities at Content > Products > Add/Edit, donation products can be configured here. Your customers will be given the ability to donate a specific amount or round their order up to the nearest dollar. You can also display multiple charities at checkout.

True sets the 'ProductisDonation' flag. False removes the flag. By default, if the column is blank or NULL, it is set false.

DonationAmtSelectedAtCheckout

Your customers can be given the ability to donate a specific amount or round their order up to the nearest dollar.

True sets the 'DonationAmtSelectedAtCheckout' flag. False removes the flag. By default, if the column is blank or NULL, column will be set to false.
