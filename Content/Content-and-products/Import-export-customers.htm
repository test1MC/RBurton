Importing and exporting customers

Use Customer Import/Export to import customer information into the site manager and export customer information to an Excel spreadsheet. You can also create a password reset email and file here.

Importing customer information automatically updates the customer database with updated information, and creates additional records for new customers. If you prefer to do this manually, you can add customers one by one on the CRM > Customers > Add/Edit page.

You should always export first to ensure you are using the most recent spreadsheet version. This will also guide you on the proper format for each column and row as well as which fields are required.
To export customer information
  1. In Content, select Import/Export > Customers.
  2. Select whether to filter customers by Create Date or Modified Date.
  3. Customer information added outside of this date range will not be pulled into the spreadsheet.
  4. Select other export options as needed:

    1. Create Email/Password Reset File – Useful when importing customers from another database. After customer information has been imported, select this option to export the customer list. This creates a file containing email addresses and encrypted unique URLs for each individual customer. The "Reset URL" strings can be used to invite your customers to reset their passwords on your new site. Additional wiring is required.

    2. Include Account Questions – Populates the Excel spreadsheet with two new fields: Q_Text and Answer_Text. To learn more about account questions, see the knowledge base article #1793. This a custom feature. Additional wiring is required.

    3. Include Address Book Entries – Adds a new row in the spreadsheet for each unique address in the address book. If there is more than one row for a particular customer, the first row is from their record (billing), and the rest of the information is from the address table. This includes all of each customer's addresses, including the billing and shipping addresses. If you don't select this option, only the billing address is returned.

    To export all customers with no address book entries—no duplicate customer information—make certain no check boxes are marked.
    When exporting in order to generate an import template, it is recommended that you select both Include Account Questions and Include Address Book Entries check boxes. This way all possible fields will be included when importing and eliminate potential discrepancies.

  5. Click the Export button .

    A link to the spreadsheet appears in the Report section beneath the Available Exports to Download area.

  6. Click the spreadsheet link to download and open the file containing the customer data. Make modifications as needed, and save it to your computer. Customer spreadsheet fields are:
Field Description
Customer_ID A unique number generated for each added customer to the platform; used internally to identify a customer record and associated information. This field should be left blank when adding a new customer. Should not be modified.
First Customer's first name. Required field.
Last Customer's last name. Required field.
Company The name of the business or company if a customer is associated with one . Required field.
Email

The email address used for customer correspondence for order receipts, RMAsreturned merchandise authorization, promotions, and other sales initiatives; the default email address for customer account login. Required field.

An email address is the primary identifier of a customer. If a customer is in the file multiple times with the same email address, the additional records are checked to determine their additional available addresses.
Password Only used when importing new customers, or if there is a reason to set a temporary password for the customer. Blank when running an export. Required field.
Street1 The customer's street number and name; used for billing purposes. Required field.
Street2 Additional address information such as suite, apartment, or P.O.Box; used for billing purposes.
City Customer's city of residence; used for billing purposes. Required field.
State Customer's state of residence; used for billing purposes.
Zip Customer's zip code of residence; used for billing purposes. Required field.
Country Customer's country of residence; used for billing purposes.
Phone Phone number associated with the customer for billing purposes. Required field.
Phone_Other Additional phone number for contact. (Evening Phone).
Phone_Mobile Customer's mobile phone number.
Fax Customer's fax number.
resalecertificate Used to import a customer resale certificate number, if applicable, for customers in the states of California, Illinois, and North Carolina. This field is located in the site manager in the Billing Address section of the retail order screen.
customerNotes Contains any notes added to the customer's account file.
create_date The date the customer account was created; should not be modified.
PlainText

Accepts only True or False values:

  • True – email correspondence to the customer will be sent in plain text only format.
  • False – email correspondence to the customer will be sent in html format.
  • If field is blank, emails will be sent as plain text by default.
SendPromo

Accepts only True or False values:

  • True – the opt-in to the mailing list for special offers and promotions check box on the CRM > Customers > Add/Edit page will be selected.
  • False – the opt-in to the mailing list for special offers and promotions check box is not selected.
SendCatalog

Accepts only True or False values:

  • True – and you have setup physical catalogs on your site, the customer will be sent a catalog to the address on file.
  • False – the customer will not be sent a catalog. If the field is left blank and you have setup physical catalogs on your site, then the customer will be sent a catalog to the address on file.
SendSMS

Accepts only True or False values:

  • True – SMSshort message service correspondence will be sent to the customer's mobile number.
  • False – SMS correspondence will not be sent to the customer's mobile number.
  • If the field is blank, SMS correspondence will be sent to the customer's mobile number by default.
Q_Text Contains questions set up for customer emails. Each question is displayed in a separate row. (Custom)
Answer_Text Contains responses to questions submitted by email. Each answer is displayed in a separate row. (Custom)
Address_ID

Important to the import process. If left blank and the email address provided is not found, the process assumes that a new customer record is being added to the system. If left blank and the email address provided is found, the process assumes that an existing customer record is to be updated.

  • If the value in the Address_ID field reads "NEW" the process will assume that this is a new address book entry for the specified customer.
  • If the value in the Address_ID field is a numeric value (provided by the export) then the process will assume that this is an existing address book entry for the specified customer and will attempt to update.
AVSMemberID The ID used to identify a customer record in with a valid address verification listing.
ExternalCustomerID Used if linking a customer account from an external source outside of Aptos .
DateOfBirth The customer's date of birth; used for age verification and for birthday-related promotions.
To import customer information
  1. In Content, select Import/Export > Customers.
  2. In the Import Customers area of the screen, click Browse and navigate to the Excel file containing the customer information you want to upload.

  3. Click the Import button to upload the file information into the site manager.
