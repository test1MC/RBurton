Adding and editing FTP hosts

Use this area to setup 3rd party FTP hosts that can be used to exchange site files between a dedicated server and Aptos for use in scheduling jobs. If a job step needs to connect to an external FTP server to import or export a file, the credentials can be modified here.

For older recurring jobs that do not use the Job Scheduler service, the FTP credentials may be stored in the import/export code. If this is the case, you will need to contact Aptos to modify the FTP credentials.

To add an FTP host

  1. From Content > Tasks > FTP Hosts, click the Add FTP Host link. A new window opens.

  2. Configure options as needed:

    Add new FTP host options
    Option Description
    Description A brief description of the FTP client being added.
    Host Name

    Name or IP address assigned by the FTP client provider.

    ftp://ftp.my-best-domain.com or ftp://68.128.255.1
    Change Directory Enter the file path that will be used to transfer files.
    User Name Assigned username provided by the FTP client provider.
    Password Assigned password provided by the FTP client provider.
    Port By default, the port is set to 21, but if the FTP server uses a different port, specify it here.
    Secure Select if using a FTPS protocol to encrypt username, password, and content during transmission.
    Passive Leave un-selected if the FTP server is set to actively listen for a dynamic port to be sent by the user client. Select if the FTP server is set to passively listen for a dynamic port after sending the source port and IP using the PASV command.
    Delete Source File Select if you want the source file to be deleted.
  3. Click Save.

About notifications

If a connection fails, you will receive an email indicating that Aptos was unable to connect to the "Host" FTP server. In the event you receive this email, log into the site manager and confirm that your credentials are correct. You can test these credentials using another FTP client, such as FileZilla. Contact the host to confirm that their FTP server is online.

If you are unable to resolve any FTP connection issues, contact your Client Manager.
