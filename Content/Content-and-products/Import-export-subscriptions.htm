Importing and exporting subscriptions

Use Subscriptions Import/Export to quickly set up multiple subscriptions at the same time. On this page, you can export and modify existing subscription properties, and set up new subscriptions.

When updating by spreadsheet, you should always export first to ensure you are using the most recent spreadsheet version. This will also guide you on the proper format for each column and row as well as which fields are required.
Exporting subscriptions
  1. In Content, select Import/Export > Subscriptions.
  2. Use the Export By lists to select to filter by:
    1. Sub-Categories or Brands, and
    2. All Available Products or Only Products With Subscriptions.
  3. Select whether to export Deleted Products and Inactive Products using the check boxes.
  4. Select categories or brands from the list on the right. To select more than one, hold down the Ctrl key. You can also click Select All to select all listed here.

  5. Click the Export to Excel button .

    This creates an Excel spreadsheet and a link to it at the bottom of the window.

  6. Click the spreadsheet link, add or update subscription information in it, and save the spreadsheet to a local drive. Spreadsheet field descriptions are:

    Subscriptions spreadsheet fields
    Field Description
    Sub-Category

    Sub-category of the current product. Read only, meant for viewing and filtering.
    Brand

    The brand of the current product. Read only, meant for viewing and filtering.
    Product_ID

    If blank, "Part_No" will be used. Not required when importing the spreadsheet.
    Part_No

    Only used if Product_ID is blank. The SKU or Part Number of the current product. Read only, meant for viewing and filtering.
    Selling Price

    The current selling price of the product. If the merchant is setting up only one subscription value, this can be copied into the Subscription_Price field. Read only, meant for viewing and filtering.
    On Sale Price

    If the current product is on sale, this price is reflected here. Read only, meant for viewing and filtering.
    Next Purchase On

    The next purchase period for the product. If the default for a purchase is to be re-bought every 30 days, then 30 would go in here.

    (Integer) The number indicates when the next purchase is set to occur. (Days = 0, Weeks = 1, Months = 2 or Years = 3). Another way to put this is: 0-Daily, 1-Weekly, 2-Monthly, 3-Annually.

    Required Field.
    Next Purchase On Period

    The next purchase will occur in how many months?

    For example, for 12 months, this number would be 12.
    Next Select List

    Comma delimited list of intervals a customer can select. If customer can choose to buy in 30, 60, or 90 days, this would be 30,60,90. The NextPurchaseOnPeriod must be within this list. These intervals appear on the cart page only if a site is using the subscription cart. In other cases, sites display these intervals on the product detail page instead.

    Required.
    Subscription Quantity

    Quantity of the product to be purchased. Indicates how many units the customer will receive. Required.
    Subscription Price

    The price that will be paid by the customer for the product when the subscription runs. Required.
    Subscription Count

    Every "X" subscriptions, the customer will get the following quantity and price of this product. Defaults to 1 for each subscription. Required.

Renewals and price breaks example

If a product price and quantity will never change regardless of how frequently the subscription renews, you simply would have a single row with Subscription_Quantity of x, Subscription_Price of y and Subscription_Count of 1.

If however, you want offer a price incentive based on the number of times a subscription renews—for example, every other purchase is 50% off, or every third purchase is free—then you would set up a row per-order interval, and set the price accordingly. For example, here's what that portion of the spreadsheet might look like if every third renewal is free.

Subscription_Quantity Subscription_Price Subscription_Count
1 22.00 1
1 22.00 2
1 0 3
Importing subscriptions
  1. In Content, select Import/Export > Subscriptions.
  1. Click Browse, find the Excel spreadsheet you want to import, and click Open.
  2. Click the Import button .

    Subscriptions are automatically uploaded to the server.
