Auto Credit task

The Auto Credit task is used to automatically issue a credit for orders with eligible returned items.

Contact your Account Manager or Client Care if you would like to configure and use this job scheduler task.

Eligible orders

Orders must meet all of these conditions for the Auto Credit task to perform:

  • Only orders paid in full with a credit card or paid in full with PayPal are eligible.
  • Only items with a Return Location indicated for the supplier ID that is holding the returned item.
  • Only items with a Returned Received status (Item Status ID = 24).
  • Only items with a valid value in Item Return Date.
  • Only items where the amount to be refunded is greater than zero.

Exclusions

  • No mixed tender scenarios.
  • If an attempt to automatically credit an item fails it will be set to Status ID = 47 and the item will no longer be eligible for the automatic crediting process.
  • Orders that have had previous credits created via the site manager or through an API are excluded because the previous tax credit on an item cannot be tracked.
  • Multiple capture scenarios are excluded as it is not possible to select the correct capture instance for crediting purposes.

Results

The credit is requested from the credit card processor or PayPal. In a successful credit, the system will perform the following actions:

  • Record and display the credit information in the Payment Information section of the Process Payment and View Order tabs of the order.
  • Record and display the item credit and tax such that the net difference is zero (0) in the Credit Order tab. It will display the portion of the tax that is credited.
  • An Order Note will be added indicating that the credit was done using the automated process and will display the Part No or Option Value, the quantity returned and the amount credited.
    • The amount credited is calculated as: Item total + item - item discount(s) - coupon discount(s). If shipping tax is attached to the item, it is also credited.

  • The item will no longer appear in the Return without Credit Report (Analytics > Order Reports > Returns without Credit).

On a failed credit, will:

  • Add an Order Note indicating an automated credit process failed along with reasons passed through from the processor.
  • The item status will be set to Auto Credit Failure (ID 47).

When an order credit is generated by the Auto Credit task job, the system will do the following:

  • If available, it will generate an affiliate level Order Credit email.
  • If an affiliate level Order Credit email template is not available, it will generate a site level Order Credit email.
