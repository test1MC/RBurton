Using the Site Provisioner

Use the Site Provisioner to create a duplicate site of an existing parent site. In addition, if more than one site exists, you may also sync data between the parent and the secondary sites. Syncing data between a secondary site and the parent site eliminates the need to update each individual site manually.

Syncing data is currently only available for site folders and a handful of logic data sets. To learn what is on the Aptos Digital Commerce road map for future syncing capabilities, contact your Client Manager.

To provision a site

When you provision a site, you are copying the exact database of the site you are currently in, including settings, product data, pages, and so on. Because of this, if you want to provision a responsive site, the base site you are provisioning from must also be a responsive site.
  1. Login to the site that you want to use as the base for the new site.

  2. Go to Content > Tasks > Site Provisioner. Make sure the Provision a Site radio button is selected.

  3. Configure options as needed:

    Site Provisioner options
    Option Description
    Site Name Enter an identifiable website and database name. This is used only for internal use—the database and content folder. Use only letters and numbers—no spaces. The name cannot begin with a number or symbol.
    Source Database Name

    The source database name that will be used to generate the new site. Select from the drop-down menu.
    Destination Parent Database If you have a site that has multiple databases (multiple brands), associate it to a Parent which can be selected from the drop-down menu at the top right of the page.
    Site URL The URL for the new site. This is the site name you want your customers to use to open your site. (This can be different from what is displaying in the field. (Use lower case letters only.)
    Binding Protocol

    Using the drop-down menu, select both binding protocols, beginning with HTTP and then followed by HTTPS. This is a two-step process:

    1. Select HTTP. The Binding Port (below) will automatically populate with 80. The Binding URL field (below) will auto-fill based upon your earlier entries. Click Add which will add it to the list under Assigned Bindings.
    2. Select HTTPS. The Binding Port (below) will automatically populate with 443. The Binding URL field (below) will auto-fill based upon your earlier entries. Click Add which will add it to the list under Assigned Bindings.
    Binding Port Automatically populates based on the Binding Protocol selection (above). Assigned binding port (for example, http=80, https=443)
    Binding URL Auto-fills based on your earlier entries. Remember to click Add to add it to the list under Assigned Bindings. As explained earlier, this is where you will add the HTTP version and then go back and select HTTPS, and add that as well.
    Assigned Bindings List of assigned bindings. There should always be at a minimum of two entries which reflect http://sub.domain.com and https://sub.domain.com.
    Notify Email Address(es) List of email addresses that will be notified when the provisioning process is complete. If you provision a site, make certain your email address is in this list.
    OMS Site Used for OMS sites only. If you are unsure, leave it unchecked.
  4. Click Save.

Syncing data between sites

Any content synced from the parent site will completely overwrite existing data on the secondary site.

Note that syncing is only available for clients with more than one site. When only one site exists, the "Save" button will not be displayed. Instead, a note will display stating that "Data sync not available for current site."

  1. In Content > Tasks > Site Provisioner, select the Sync Data Between Sites radio button.

  2. Configure options as needed:

    Sync Data Between Sites options
    Option Description
    Select Site to Sync Content From: Select the parent site that you would like to sync data from. (The site that you are currently logged into will be the destination site where the data will be imported.)
    Notify Email Address(es):

    Enter the email addresses that you would like to be notified when a site has been synced. (Delimited by commas or semi-colons).
    Schedule Sync for (CST): Schedule a date and time for a sync to occur. This will automate the syncing process and can be used to minimize disruption due to site data loading. Note that the time entered here will be recorded as CST - the timezone where the servers are located.
    Select Available Content to Sync: Select the items that you would like the site to sync data.
  3. Click the sync button to begin data synchronization.
