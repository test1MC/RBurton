Verify information is correctly processed

Monitor the movement of data to make sure correct values are stored in InfobaseInfobase, the Aptos transactional database, is a collection of tables that stores the results of merchandising transactions. tables.

Coordinate system processes with physical inventory

Run tasks before, during, and after counting physical inventory, to post inventory counts and ensure shrink is calculated correctly.

Maintain entities and data tables

Employees, locations and vendors are the data records most likely to require regular maintenance, along with tables that are referenced by data records. You can run periodic tasks for styles.

Maintain store databases by updating PLU files

Run an update when changes affect your location hierarchy, merchandise hierarchy, or prices. You can run updates for one, some, or all store databases, depending on the changes.

Define, run and monitor processes

Most Merchandising processes, including data import and export, exchange data among applications, statistical tracking, and volume reporting, are scheduled and run with the Pipeline tool. Some scheduled jobs in POM and A&R require you to use the Foundation Server Monitor to create jobs, set up schedules for them, and ensure that they run correctly.

Modify parameters

Have the flexibility to change your system parameter settings to handle changing business circumstances. This is generally done in consultation with Aptos.
