Preparing for inventory counts

Gather this information

To prepare for an inventory count, find out these details:

  • merchandise to count
  • locations where the count will occur
  • valuation date
  • count date
  • count update type
  • formats to use for importing count files.

You require this information to create inventory control documents (ICDs) in Inventory Management. See for more about ICDs.

Updates to styles and current retail

Any styles that are to be counted must be included in the merchandise hierarchy on or before the count date.

Make sure that any permanent retail price changes to be in effect on or before the count date are in effect on or before the effective date.

You add and update styles in Product Management. See .

Physical movement of inventory

In Inventory Management verify these items so that the book inventory is as accurate as possible before you calculate and evaluate shrinkage.

PO and Unsolicited Receipts

If the status of these is Received, no action is required. The items have updated on hand quantities and can be included in the count. If goods have arrived in the warehouse or receiving store and are to be counted, change their status to Received so that they are not double counted.

Store Shipments

If the status is Sent because the goods have not yet arrived in the receiving store, no action is necessary. If goods have arrived in the receiving store and will be counted, change their status to Received so that they are not double counted.

RTVs

If the status is Returned, no action is necessary. If the status is Requested because authorization has not yet come from the vendor, include these units in the count file or adjust the count before you run a preliminary shrink report.

Shrinkage and User-Defined Adjustments

If the status is Submitted, no action is required. If the status is Preliminary, do not include them.

Transfers

If the status is Received, no action is necessary. If the status is Sent because the goods have not yet arrived in the receiving store, no action is necessary. However, if the goods arrive in the receiving store prior to the inventory date, do not be count them until they are received in Inventory Management.

See the relevant chapters in Inventory Management User Guide for detailed information about these documents.

On hand inventory report

Create a view in Query Designer that shows on hand inventory by status. This captures pre-count values that would otherwise be lost, and helps find areas where inventory might be overlooked or adjustments need to be made. Posting the count clears Unavailable – Discrepancy and Unavailable – Pending Shrink. The report retains a record of these values.

Posting the count does not clear user defined Unavailable statuses.

Prior to the count and or after posting, some user-defined Unavailable inventory statuses can require adjustment to prevent them resulting in a negative number of units with Available status. The report helps find these potential adjustments. See for an example of this kind of adjustment.

If unavailable goods are physically absent from a count location, they can still be part of its inventory. For example, items with the status Unavailable – Out for Repair must be included in the count even though they are in a repair shop. The report helps you identify such items.

Count import definitions

If your company uses a third party to count merchandise, they can use different formats for count files. In that case, you create or modify count import definitions before you import count information into the system.

File formats for importing inventory counts are defined in Enterprise Data Management. A file format identifies the fields in the file and the order in which they appear. There are two file formats, fixed and delimited.

About fixed formats

A fixed format means that fields in the file are in a specific order and that each field occupies a fixed number of positions in the record. For example, the location code can always be in positions 1 to 4 of a record, UPC is always in positions 5 to 18, and units counted of the UPC will are in positions 19 to 23.

Field

Description

Code

Required. An identifier assigned to the definition. Alphanumeric, can be up to 8 characters.

Description

Text describing the format so that a user knows which one to select.

Import type

Fixed or Delimited, as selected when creating the count import definition.

Active

Indicates that this entry is active. Selected by default.

Type

The field whose location within the record is being specified. Fields are entered in the order they appear in the file. Options are:

Cost: For regular styles, the cost per unit counted. For pseudo styles, the total cost counted.

Location: Location code of the store or warehouse the count is being done for.

Retail: For regular styles, the retail per unit counted. For pseudo styles, the total retail counted.

UPC: numeric code that uniquely identifies the merchandise at the style/color/size level.

Units: Number of units counted of the UPC at the location.

Zone: Area in the location being counted, such as front of store, window display, or stockroom.

Start Position

Position number of the first character of the associated field.

Length

Number of characters the field will take in the record.

To define a fixed file format

  1. In Enterprise Data Management select General > Inventory > Count Import.
  2. Click Document > New.
  3. In Import type select Fixed > OK.
  4. Enter a reference Code and Description for the file format.
  5. In Type select the first field that will appear in the record, and enter its Start Position and Length.
  6. Repeat for each field in the record.
 About delimited formats

A delimited format means that the fields in the file are in a specific order but each field can be a different length in each record. To separate fields without knowing how long each field will be, you specify a delimiter, such as a comma, to signify the end of the field.

If a location code contains leading zeros (such as the 5‑digit code 00237), the leading zeros must be included in both the field and the import definition.

In Enterprise Data Management, select General, Inventory, Count Import.

Click Document > New.

In Import type, select Delimited>OK.

Enter a reference Code and Description for this file format.

Select a Delimiter to use as a separator between fields in each record.

For Type, select the fields in the order in which they will appear in the record.
