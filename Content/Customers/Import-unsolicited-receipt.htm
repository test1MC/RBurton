Importing unsolicited receipts

  1. Create an XML text file that conforms to the file layout.
  2. Run the Merchandising.Import XML Documents Services Administration job. For information on running jobs, see the Job Management Reference.

Notes

Action

Attribute

String

Required.

“add” creates a new document.

“update” edits a document.

“delete” removes a document.

EntityAction

Attribute

String

"Receive" sets the document status to Received which will update IB Inventory.

"Cancel" deletes the receipt units from IB inventory.

"Reinstate" reverts a cancelled receipt to preliminary status. A new receive transaction is then needed to add the receipt units back to inventory.

"PostToInvMatch" used when IMAT is turned off. This is the only way for a match status to be updated from prelim to unmatched.

Attributes

Attributes are optional. Multiple attribute records for the same unsolicited receipt are permitted so long as the rules of exclusivity with respect to any associated attribute sets are respected.

Messages

Messages are optional. Multiple messages for the same unsolicited receipt are permitted.

Tag Name

Tag Type

Data Type

Notes

Action

Attribute

String

Required. Value can be "add".

Adds a message to a unsolicited receipt document.

Actions are case sensitive and must be enclosed in quotes.

MessageTypeDescription

Element

String

Required. Must be an active message type created for Inventory Management.

If the message type is exclusive, then it must be unique within the unsolicited receipt document (i.e. the same message type can only be added once to the same unsolicited receipt document).

MessageCode

Element

String

Required if MessageTypeDescription is not provided.

Provide this field if a predefined message code (as defined in EDM > General > Messages) is to be added to the unsolicited receipt.

If provided, must be an active message code that belongs to your MessageType.

MessageText

Element

String

Mandatory if MessageCode was not provided.

Only provide this field if a free-form message is to be added to the unsolicited receipt.

The message must adhere to the maximum length restriction of the message type that was selected (as defined in EDM > General > Messages).

A message can be provided using a predefined message code for the message type or you can provide a free-form message text. If both have been provided, then the system will update the unsolicited receipt with the free-form text (i.e. MessageDescription).
