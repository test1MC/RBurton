Importing size scale information

  1. Create a comma or tab-delimited text file that conforms to the file layout.
  2. Run the AR.Size Scale Import Services Administration job. For information on running jobs, see the Job Management Reference.

Notes

Create the input files in the order they need to be processed. The import sorts and processes the files in a folder based on the file creation date. Using the creation date lets the import job control the sequencing and avoid timing errors.

If generate level = merchandise group, provide a scale value for each size code assigned to the size category. If not provided, the scale value is 0 for any size not provided.

If generate level= style, provide a scale value for each size code assigned to the style. If not provided, the scale value is 0 for any size not provided.

If generate level= color/size, provide a scale value for each color/size assigned to the style. If not provided, the scale value is 0 for any color/size not provided.

If importing a record that already exists, then existing record is overwritten with the new imported record.

The file layout must include two row types: Header and Detail. The user will provide one Header record for each distinct size scale. Each Header record contains 1 to many detail records.

At least one detail record must exist for a header record. Each Header record will contain 1 to many detail records. The detail records have to be sequenced to follow the Header record.
