Importing transfers

  1. Create an XML text file that conforms to the file layout.
  2. Run the Merchandising.Import XML Documents Services Administration job. For information about running jobs, see the Job Management Reference.

File layout

XSD definition

The XSD file is installed with Merchandising and is called Transfer Import.xsd.

Action

Attribute

String

“add”

“update”

“delete”

Required.

“add” will create a new document.

Note, if “add” is the action and the EntityAction is not provided, then this will create a Transfer in preliminary status.

“update” will modify an existing document. When update is used, you must provide the document number.

“delete” will delete an existing document. When delete is used, you must provide the document number.

EntityAction

Attribute

String

“Send”

“Receive”

"Cancel"

"Reinstate"

“SetReadyToSend”

“Send": Can only use this entity when action is add. This will create a new transfer in Sent status. Merchandise is removed from the sending location’s inventory and added to the receiving location’s intransit inventory.

"Receive": Can only use this entity when action is update. This updates a transfer from Sent to Receive status. The receiving store inventory is moved from the intransit status to the available status.

"Cancel" will cancel a transfer in ‘sent’ status. Can only use this entity when action is update. The quantities will be backed out from receiving location’s inventory and added back into the sending location’s inventory. A cancel can also be used to cancel a ‘ready to send’ transfer.

“Reinstate” will revert a canceled document back to preliminary status. Can only use this entity when action is update.

“SetReadyToSend” " Used to advise store staff that a transfer created by head-office is ready to be sent. As an import, this is NOT relevant in Standalone A&R scenarios.

XML examples

To create and send a transfer

<?xml version="1.0" ?>

<Transfer CompanyId="1" Action="add" EntityAction = "Send">

<DocumentNo>900001</DocumentNo>

<DocumentSource>3</DocumentSource>

<FromLocationCode>0044</FromLocationCode>

<ToLocationCode>0055</ToLocationCode>

<ReasonCode>EXTTR</ReasonCode>

<SKUs>

<SKU Action="add">

<UPCNo>400000008349</UPCNo>

<UnitsSent>5</UnitsSent>

</SKU>

</SKUs>

</Transfer>
To receive a transfer
<?xml version="1.0" ?><Transfer CompanyId="1" Action=" update" EntityAction = "Receive"> <DocumentNo>900001</DocumentNo> <SKUs> <SKU Action="update"> <UPCNo>400000008349</UPCNo> <UnitsReceived>5</UnitsReceived> </SKU> </SKUs></Transfer>
To modify a transfer
<?xml version="1.0" ?><Transfer CompanyId="15" Action="update"><DocumentNo>000298</DocumentNo><DocumentSource>1</DocumentSource><FromLocationCode>0033</FromLocationCode><ToLocationCode>0044</ToLocationCode><ReasonCode>T01</ReasonCode> <SKUs> <SKU Action="update"> <SKUId>200006</SKUId> <UnitsSent>20</UnitsSent> </SKU> </SKUs></Transfer>
To cancel a transfer
<?xml version="1.0" ?><Transfer CompanyId="15" Action="update" EntityAction = "Cancel"> <DocumentNo>1000000</DocumentNo></Transfer>
