Importing PO receipts

  1. Create an XML text file that conforms to the file layout.
  2. Run the Merchandising.Import XML Documents Services Administration job.

For information about running jobs, see the Job Management Reference.

File layout

XSD definition

The XSD file is installed with Merchandising and is called PO Receipt Import.xsd.

 File structure definition

Header

Tag Name

Tag Type

Data Type

Notes

Action

Attribute

String

Required

“add” creates a new document.

“update” edits an existing document.

“delete” removes a document.

EntityAction

Attribute

String

“Receive” adds receipt units to inventory.

“Cancel” removes the receipt units from inventory.

“Reinstate” reverts the receipt status back to preliminary status. A new receive transaction is then needed to add the receipt units back to inventory.

“PostToInvMatch” is applicable if Invoice Matching is not installed.

"Ship" is applicable if the PO Receipt is generated from an ASN and Track in-transit is enabled for the vendor.

DocumentSource

Short

DocumentDescription

Element

String 60

 

MatchStatus

Element

Short

Must be set to 1 (preliminary) if the client plans to send modifications/cancellations for the PO receipt. In this case, pipeline segment 3008 must be run which will set prelim matched receipts older than X days to ‘fully matched’, this is required for the system delete process to remove old receipts. Alternatively a PO Receipt modify for the match status can be sent when the PO receipt is completed processed (the modify would be to set the match status to fully matched).

Can be set to 5 (fully matched) if no modifications/cancellations will be posted against the receipt.

Can be:

0 (undefined)

1 (preliminary)

2 (unmatched)

3 (partially matched

4 (over matched)

5 (fully matched)

Mandatory if the EntityAction is "Receive"

This is existing functionality in pre-ME5.0 SP4, it is not changed due to the feature. The IDMD needs to be updated to reflect this.

Element

String 5

ShipViaCode

Element

String 2

If provided, must be an active ship via code predefined in EDM.

If the PO Receipt will be linked to a PO Expected Receipt Date, set the Ship Via Code equal to the Shipment Ship Via Code so that the Ship Via that is linked to the PO Expected Receipt Date is displayed on the PO Receipt - General tab - Ship via field.

This field is modifiable.

AdvanceShippingNoticeNo

Element

String 20

ASN number if the PO receipt was generated from an ASN.

Mandatory for PO Receipt in "Shipped" status.

CarrierCode

Element

String 4

If provided, must be an active carrier code predefined in EDM.

If the PO Receipt will be linked to a PO Expected Receipt Date, set the Carrier Code equal to the Shipment Carrier Code so that the Carrier that is linked to the PO Expected Receipt Date will be displayed on the PO Receipt - General tab - Carrier field.

This field is modifiable.

AllocationReplacedFlag

Element

Boolean

Not applicable. Do not provide.

Can be False or True

ImatchWarningGivenOrIgnored

Element

Boolean

Not applicable. Do not provide.

Can be False or True.

ShipDate

Element

Datetime

Mandatory for PO Receipt in "Shipped" status. If not provided, set to Current Date.

Format: yyyy-MMddTHH:mm

ShipmentExpectedReceiptDate

Element

Date

If provided, it must be an expected receipt date that exists on the PO.

Modifiable if PO Receipt status is "Preliminary".

If the PO Receipt must be linked to a PO Expected Receipt Date, then this field must be provided.

Format: yyyy-MM-dd

ShipmentShipViaCode

Element

String 2

Identifies PO Shipment level Ship Via Code.

If provided, it will be used to identify the correct PO Shipment if there are duplicate PO Expected Receipt Dates on the PO.

The value provided in this field will NOT be displayed on the PO Receipt - General tab - Ship via field.

This field is NOT modifiable.

 

ShipmentCarrierCode

Element

String 4

Identifies PO Shipment level Carrier Code.

If provided, it will be used to identify the correct PO Shipment if there are duplicate PO Expected Receipt Dates on the PO.

The value provided in this field will NOT be displayed on the PO Receipt - General tab - Carrier field.

This field is NOT modifiable.

ShipmentCountryCode

Element

String 3

This identifies the PO Shipment level Country Code.

If provided, it will be used to identify the correct PO Shipment if there are duplicate PO Expected Receipt Dates on the PO.

The value provided in this field will be displayed on the PO Receipt - General tab - Country if it matches with the Country that is linked to the PO Expected Receipt Date.

This field is NOT modifiable.

ShipmentFOBDescription

Element

String 20

This identifies the PO Shipment level FOB description.

If provided, it will be used to identify the correct PO Shipment if there are duplicate PO Expected Receipt Dates on the PO.

The value provided in this field will be displayed on the PO Receipt - General tab - FOB if it matches with the FOB that is linked to the PO Expected Receipt Date.

This field is NOT modifiable.

XML examples

To create and receive a PO receipt

<?xml version="1.0" ?>

<POReceipt CompanyId="15" Action="add" EntityAction = "Receive">

<DocumentNo>1000002</DocumentNo>

<DocumentSource>3</DocumentSource>

<ReceiveDate>2013-07-26T00:00:00</ReceiveDate>

<LocationCode>0011</LocationCode>

<PONo>102696</PONo>

<SKUs>

<SKU Action="add">

<StyleCode>100000001</StyleCode>

<ColorCode>001</ColorCode>

<SizeCode>NoSz</SizeCode>

<UnitsReceived>2</UnitsReceived>

</SKU>

</SKUs>

</POReceipt>
To modify a SKU receipt
<?xml version="1.0" ?><POReceipt CompanyId="15" Action="update"> <DocumentNo>1000002</DocumentNo> <SKUs> <SKU Action="update"> <StyleCode>100000001</StyleCode> <ColorCode>001</ColorCode> <SizeCode>NoSz</SizeCode> <UnitsReceived>4</UnitsReceived>
</SKU> </SKUs></POReceipt>
To modify a pack receipt
<?xml version="1.0" ?><POReceipt CompanyId="15" Action="update"> <DocumentNo>101110</DocumentNo> <SKUs> <SKU Action="update"> <PackCode>000001</PackCode> <UnitsReceived>5</UnitsReceived> </SKU> </SKUs></POReceipt>
To cancel a receipt
<?xml version="1.0" ?><POReceipt CompanyId="15" Action=" update" EntityAction = "Cancel"> <DocumentNo>100005</DocumentNo></POReceipt>
To create and ship a PO receipt

<?xml version="1.0" ?>

<POReceipt CompanyId="15" Action="add" EntityAction = "Ship">

<DocumentNo></DocumentNo>

<DocumentSource>3</DocumentSource>

<LocationCode>0011</LocationCode>

<PONo>104825</PONo>

<AdvanceShippingNotice>000608</AdvanceShippingNotice>

<ShipDate>2014-07-26T00:00:00</ShipDate>

<SKUs>

<SKU Action="add">

<StyleCode>100000001</StyleCode>

<ColorCode>001</ColorCode>

<SizeCode>NoSz</SizeCode>

<UnitsReceived>0</UnitsReceived>

</SKU>

 

<SKU Action="add">

<PackCode>000052</StyleCode>

<UnitsReceived>26</UnitsReceived>

</SKU>

</SKUs>

</POReceipt>
To ship a receipt
<?xml version="1.0" ?><POReceipt CompanyId="15" Action=" update" EntityAction = "Ship"> <DocumentNo>101970</DocumentNo></POReceipt>
To create and receive a PO receipt that is linked to a PO Expected Receipt Date, and PO Shipment Level Vendor

<?xml version="1.0" ?>

<POReceipt CompanyId="15" Action="add" EntityAction = "Receive">

<DocumentNo></DocumentNo>

<DocumentSource>3</DocumentSource>

<ReceiveDate>2015-06-23T00:00:00</ReceiveDate>

<LocationCode>0011</LocationCode>

<PONo>104825</PONo>

<ShipViaCode>AE</ShipViaCode>

<ShipmentShipViaCode>AE</ShipmentShipViaCode>

<ShipmentCarrierCode>5022</ShipmentCarrierCode>

<ShipmentExpectedReceiptDate>2015-09-30</ShipmentExpectedReceiptDate>

<ShipmentCountryCode>UK</ShipmentCountryCode>

<ShipmentFOBDescription>Vancouver</ShipmentFOBDescription>

<SKUs>

<SKU Action="add">

<StyleCode>100000001</StyleCode>

<ColorCode>001</ColorCode>

<SizeCode>NoSz</SizeCode>

<UnitsReceived>0</UnitsReceived>

</SKU>

<SKU Action="add">

<PackCode>000052</StyleCode>

</SKU>
