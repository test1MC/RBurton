Importing store shipments

  1. Create an XML text file that conforms to the file layout.
  2. Run the Merchandising.Import XML Documents Services Administration job. For information on running jobs, see the Job Management Reference.

File layout

XSD definition

The XSD file is installed with Merchandising and is called Store Shipment Import.xsd.

 File structure definition

<StoreShipment>

header elements…

<SKUs>

<SKU>

detail elements...

</SKU>

</SKUs>

</StoreShipment>
Header

Tag Name

Tag Type

Data Type

Notes

Action

Attribute

String

Required.

“add” creates a new document.

If action = add and no entity action is specified, the store shipment will be created in the Sent status if the receiving location’s flag for auto-receive shipments is set to False. If the receiving location’s auto-receive shipment flag is set to True then the store shipment is created in the Received status.

“update” edits a document. When update is used, you must provide the document number.

“delete” deletes a document. When delete is used, you must provide the document number.

EntityAction

Attribute

String

"Receive" can only be used against the update action.

Receive along with the update action will update the document’s status from Sent to Received.

Only provide this field for location’s who flag for auto-receive shipments is set to False.

CompanyId

Attribute

Integer

Required. Company number that the document resides.

DocumentNo

Element

String 20

Required. Store shipment number.

For an add action, this is not mandatory as the system will generate the document number if one is not provided.

Mandatory if action is update or delete.

ReceiveDate

Element

Datetime

The actual date the receiving location received the merchandise. This can only be sent with a receive transaction. Format: yyyy-MMddTHH:mm

ShipDate

Element

Datetime

Required. Date the shipping location shipped the merchandise. Format: yyyy-MMddTHH:mm

ExpectedReceiptDate

Element

Datetime

Required. Date the merchandise is expected to arrive at the receiving location. Format: yyyy-MMddTHH:mm

PerformedBy

Element

String 60

Name of the user who shipped the merchandise.

ExternalSystemName

Element

String 20

Name of the system that created the store shipment.

Weight

Element

Double (15,3)

The weight of the shipped merchandise.

Mandatory if the unit weight code has been provided.

UnitWeightCode

Element

String 10

Mandatory if the weight has been provided.

UnitWeightId

Element

Byte

Database reference to the unit weight code.

ToLocationCode

Element

String 20

Required. Location that the merchandise is being shipped to.

ToLocationId

Element

Short

Database reference to a location.

FromLocationCode

Element

String 20

Required. Location (warehouse) the merchandise is being shipped from.

FromLocationId

Element

Short

Database reference to a location.
SKU details

Tag Name

Tag Type

Data Type

Notes

Action

Attribute

String

Required.

“add” will add a new detail line to existing document.

“update” will modify an existing detail line of a document.

“delete” will remove a detail line from a document.

UnitsReceived

Element

Long

Number of units received by the receiving store. Mandatory if the entityaction is receive. Must be >=0.

UnitsSent

Element

Long

Required. Number of units shipped to a receiving location. Must be >=0.

CartonNo

Element

String 20

Not applicable.

DistributionNo

Element

String 20

Required. Distribution for which the store shipment was created for.

SKUId

Element

Decimal

Database reference to a style/color/size.

UPCNo

Element

String 14

Required if style is not provided.

StyleCode

Element

String 20

Required if UPC code is not provided.

ColorCode

Element

String3

Required if style code is provided and style is colored.

PrimarySizeLabel

Element

String 8

Required if style code is provided and style is sized.

SecondarySizeLabel

Element

String 8

Required if style code is provided and style is 2-dim sized.

XML examples

To create and send a store shipment

<?xml version="1.0" ?>

<StoreShipment CompanyId="15" Action="add">

<ShipDate>2010-08-16T00:00:00</ShipDate>

<ExpectedReceiptDate>2010-08-16T00:00:00</ExpectedReceiptDate>

<FromLocationCode>0012</FromLocationCode>

<ToLocationCode>0011</ToLocationCode>

<SKUs>

<SKU Action="add">

<SKUId>15600087</SKUId>

<DistributionNo>004441</DistributionNo>

<UnitsSent>10</UnitsSent>

</SKU>

</SKUs>

</StoreShipment>
To receive a store shipment
<?xml version="1.0" ?><StoreShipment CompanyId="15" Action="update" EntityAction = "Receive"><DocumentNo>0000000907</DocumentNo><ReceiveDate>2010-08-17T00:00:00</ReceiveDate> <SKUs> <SKU Action="update"> <UPCNo>400000029719</UPCNo> <UnitsReceived>5</UnitsReceived> </SKU> </SKUs></StoreShipment>
To modify a store shipment

Only received units can be modified.

<?xml version="1.0" ?><StoreShipment CompanyId="15" Action="update"><DocumentNo>0000000908</DocumentNo> <SKUs> <SKU Action="update"> <UPCNo>400000029719</UPCNo> <UnitsReceived>3</UnitsReceived> </SKU> </SKUs></StoreShipment>
