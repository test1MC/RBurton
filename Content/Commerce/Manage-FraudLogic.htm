Managing FraudLogic

FraudLogic enables you to configure your site's fraud settings during the checkout process to allow for an automated reviewing process. This functionality will help to reduce the overall business risk associated with fraudulent transactions. With FraudLogic, rules can be set to mark only certain orders for review, eliminating the need to manually review all orders. In addition, you have available even more protection to prevent fraudulent orders from being fulfilled with Advanced Fraud Scoring.

Advanced Fraud Scoring

If your site does not use Advanced Fraud Scoring, skip to Setting up FraudLogic Basics to learn about setting up FraudLogic. If you are interested in Advanced Fraud Scoring, review information presented here, or contact client services.

With our latest advancements in fraud scoring, you have the ability to score each order as it comes in from both credit cards and PayPal. This advanced fraud functionality gives you added protection from fraudulent activity on your website. You can effectively identify and review orders to make sure both you and your customers are protected.

Advanced fraud scoring is available to all clients. However, the solution you choose may require additional set-up fees and monthly fees, plus any applicable vendor fees. For more information, contact client services. Advanced Fraud Scoring is enabled at Users > Website Settings > General Settings > Aptos Server Settings.
Handling more sophisticated fraud techniques

Unfortunately, card-not-present online fraud is becoming more prevalent. Traditional fraud screening tools can only determine if a credit card is legitimate or if the user-entered account information matches those on record. In fact, fraudsters can obtain personal credit card information, pose as the legitimate cardholder, and bypass standard fraud checks.

By enabling advanced fraud scoring, the system examines an online transaction from all the necessary view points. Our tools verify both the authenticity of the credit card details used for the purchase, and whether the purchaser is the legitimate card holder. FraudLogic is able to identify traits and patterns that are associated with fraudulent orders. By asking the right questions, the system passes on the necessary information to detect fraudulent orders before the payment is processed.

If Advanced Fraud Scoring has been set up at Users > Website Settings > General Settings > Aptos Server Settings, there are three options you need to configure at FraudLogic Advanced Scoring.

FraudLogic Advanced Scoring settings
Option Description

FraudScore Threshold

Set to the number required for your business needs. This number can be from 1 to 100. This number represents the estimated probability that the order is fraud, based on analysis of past transactions.

If you only want to trigger advanced fraud review for orders over a certain amount, set this value to the minimum triggering amount.

If you only want to use Advanced Fraud Scoring for international orders, select this check box.

Viewing fraud score results

When set up, to help you further determine the actual fraud score threshold you want to use for your site, you can view the results on a per-order basis. Go to Commerce > Orders > View to view the order. In the Order Notes section, you'll see that all orders, regardless of threshold, store the fraud scoring results in the order notes.

Order Status Settings and Fraud Review

Order Notes

Order Notes are written for each fraudulent item within an order. This functionality checks for fraudulent items that may be associated to an order and outputs all items that are selected. Instead of checking for one and setting the order as fraudulent, the order will contain note(s) for all the reasons an order was flagged by FraudLogic. This enables you to be aware of all possible fraudulent issues that may be associated with a particular order.

Fraud Review

When you complete fraud review for an order with the status of "Processed - Pending Review" and determine that the order is valid and should be fulfilled, the order can be released into the normal workflow in one of two ways. To learn more about order statuses, go to Commerce > Orders > Manage Existing.

To release multiple orders,
  • Select the order check boxes and select a value from the Batch Process Option drop-down menu at the bottom of the Commerce > Orders > Manage Existing page. This will allow for automated processes, Quick Ship, and other activities to commence by selecting one of these options in the batch processing setting for all checked orders.
 To release an individual order,

  • Click Remove Fraud Hold on the View Order tab in the order details window accessible on the Commerce > Orders > Manage Existing page, illustrated directly below. The resulting order status will be determined based on the order status selected from the Order Status Settings drop-down menu on the FraudLogic screen.

 CVV Security Code Settings

EOM allows for specifications on how you would like to process credit card transactions when the CVVCard Verification Value Code entered by a customer during checkout is not correct. The credit card payment processors will generally return a Match/No Match response based upon the CVV Code entered.

Select the Fail transactions during checkout that have a CVV Code Mismatch box to enable this functionality. If the gateway returns back a 'No Match' response, the system can void the transaction and return a friendly message to the customer allowing them to retry within the given number of retries allowed. Please note that the response check does require a full authorization to be completed and then voided.

This feature is currently only available for First Data, CyberSource, and Authorize.net. If you have further questions, please contact client services.

Additional Validation

The additional credit card validation available on the checkout page prior to authorization includes:

  • Prompts the customer to correct entry in the event of a check digit error or card type mismatch.
  • CVV length mismatch (4 for American Express, and 3 for all others).
  • Invalid expiration date handling if the selected expiration date is in the past.

There is further validation on the checkout page to ensure that the credit card information entered by the customer in the checkout is accurate before the form is submitted.

 AVS settings

AVS Address Verification System can be your first line of defense against fraudulent transactions. To make the most of this system it is important to understand how it works.

AVS attempts to match the billing address entered as part of ordering process against the address listed on the credit card statement, as recorded by the Visa/MC Interchanges. To overcome the problems with abbreviations, misspelling, etc., the AVS strips away all letters and attempts to match only the numbers in the street addresses and the zip code. No street names or the name of the state are checked during this process.

The AVS information is only obtained back from the processor after an authorization is performed. If you wish to reject those transactions which fail to meet the desired AVS matching level, then the system will perform an immediate void of the transaction, inform the customer that their address did not match, and allow the customer to enter a new address. Since an authorization has already occurred, most payment processors will assess their per transaction fee for this authorization even though the transaction was voided.

Because AVS information is only available on credit cards issued through US banks, if you wish to accept international orders, you must select a less stringent matching rule for these orders or the international orders will be rejected.

AVS Response Codes

You can customize the AVS settings on your site for each of the major credit cards. You can get very specific when enabling these settings. You can choose from multiple options for each card on items such as: zip code does not match the address, no data from issuer/BankNet switch, ZIP code matches but the address does not, and so on. Also, this is a good way to enable even more strict settings when accepting international orders.

Fraud codes are configurable based upon the payment processor that is selected. The settings for the AVS Response Codes are pulled from the listing tables fully configurable at CRM > Communications > Listings.
