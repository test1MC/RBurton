Configuring fulfillment rules

Use order routing to easily control the order of priority for suppliers who fulfill orders.

Understanding and applying order routing

Configure order routing to prioritize the rules applied to all suppliers when fulfilling orders. The higher in the order a rule is set, the more weight it receives when the system determines which supplier will receive the order.

While the actions of each rule can't be edited, you can drag and drop rules to enable them or deactivate them. You can also arrange the order in which you want the rules to be applied.

You can set the maximum number of supplier splits that can occur on any one order using Order Routing, which can give you more control over your shipping costs. This is set up in the Order Routing job, Custom4 (see Content > Tasks > Job Scheduler).

  • A setting of '0' means the entire order must be fulfilled by one supplier.
  • '1' means the order can have one supplier split, and so on. If the Order Routing job Custom4 field is left blank, unlimited supplier splitting is allowed.

When order routing cannot be completed within the allowed number of supplier splits, all items in the order are unable to be fulfilled and an order note will indicate "Unable to Fulfill - Cannot fill order within maximum allowed number of splits".

Note about Stores: Individual store locations are treated as Suppliers for order fulfillment. Stores are treated as Affiliates for order origination, and as Locations for store locator pages.

Set up suppliers before configuring your fulfillment rules. Suppliers are configured at Commerce > Suppliers > Add/Edit. After fulfillment rules are configured here, you must set up a Toro job at Content > Tasks > Job Scheduler to enable the functionality

To configure fulfillment rules

  1. In Commerce, select Suppliers > Order Routing.

    All Available Rules are displayed in the left column.

  2. Drag and drop rules into the Applied Rules column. You can move items back and forth between columns as needed. You can adjust the priority in which a rule is applied by dragging it up or down in the Applied Rules column.
  3. Some rules require that you set parameters for them after you add them to the Applied Rules column. For example, when you add the Distance rule, a Distance parameters item is added to the Rule Settings column on the right in which you can add Distance Ranges.
  4. Click Save to apply the settings.

Category Title Description
  Priority Suppliers have a priority value. By default, this is the top rule.
  Payment Terms The payment terms between your company and the supplier.
  Order Queue How many pending orders each supplier has.
Dropship Shipping Cost The cost of shipping an order.
Inventory

 

 Left Over Inventory How many items are still in the supplier's inventory after an order is completed.
Order Volume The total volume going to each store.

Rank

 Based upon the total number of items in each order. The system looks at the inventory of each supplier. For example, if an order contains 5 items, all suppliers that can fulfill the entire order will get a value of 5. Suppliers who can partially fulfill the order, will get ranked according to how many items in the order they can ship. Note that the ranking changes for each order depending upon how many items are in the order.

Sku Velocity

How many of each SKU a supplier has entered within a given timeframe. Available via a spreadsheet.

Shipping

 

Distance

The distance of the supplier from an order. When Distance is moved from the Available Rules column to the Applied Rules column, the Rules Settings column is displayed in which you can enter configurable ranges for distance.

When configured, the system treats all suppliers within a distance range as equal. for example, if the first distance range is from 0-50 miles, then any suppliers that are 0-50 miles away from the ship-to address are treated equally and given first priority. You can configure up to 10 ranges. Click the +Add Range button to add more fields. The high value is not required on the last range configured.

All suppliers in the same distance bucket are treated equally. For example, if the rules are set in this order on your site: Priority, Rank, Distance, Left Over, Order Volume, and SKU Velocity, the suppliers will be sorted by Priority first (note all suppliers with priority = 1 listed first), Rank second (all suppliers with an equal priority will then be sorted by rank), Distance third (all suppliers with equal rank will then be sorted by distance bucket value), then by left over inventory fourth (all suppliers with the same distance bucket value will then be sorted by left over inventory (which sorts description), and so on.
