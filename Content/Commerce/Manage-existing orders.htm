Managing existing orders

Use this section of the Site Manager to see an order's status at any time during the order cycle, from the status of the order in general, to exactly when the order was placed. The information shown gives you a valuable snapshot of how well your products are selling, and tells you who’s buying them. You can also process edits, payments, re-orders, returns, credits, POs, and shipments, all from one place.

To manage an existing order

1. Find the order
  1. In Commerce, select Orders > Manage Existing.

  2. Enter search information in any of the search fields: Order Number and/or dates. You can also add an additional search field by clicking Add Search Field, and selecting one you'd like to use.

    To search with even more options, click Advanced Search to display additional search criteria.

  3. Click Search.

    Search results appear at the bottom of the page.

    All columns (except Order Number) are sortable:

    • Date - Date and time stamp of the order.
    • Order Number - Access an order by clicking on the order number link. The order number is displayed in the button.
    • Order Information - This column contains only the most pertinent information about the order; customer, source, and location.
    • Type - the delivery type for the order.
    • Order Status - click (+) to open and view more details about the status of items within an order. Click (-) to close the details.
    • Total - the order total.

Single orders

Click on an order number to manage a single order. There are many features on the Managing Orders page you can use to quickly view, manage, and orders. These features are described in Using the view order window.

Batch processing orders

There are many options you can use to batch-process orders. Find the batch process options located at the bottom of the window.

  1. Use the orders' check boxes in the left-most column to select orders on which you want to run a batch process.
  2. Select the type of batch processing you want from the list at the bottom of the window.

  3. Click (>) Continue to proceed.

Batch process options
Option Description
Capture Funds Captures the funds of the selected orders.
Delete Orders Marks and then batch deletes orders.
Export Orders Exports all selected orders.
Generate Mailing Labels Generates mailing labels for the selected orders.
Generate Packing Slip Generates packing slips for the selected orders. Barcodes are displayed on packing slips, making for easier tracking from the packing slip.
Generate Pick List Batch process pick lists.
Mark as API Exported Makes the selected orders unavailable for API Export. The export status changes to API Exported.
Mark as API Unexported Makes the selected orders available for API Export. The export status changes to API Unexported.
Mark Incomplete Items as API Unexported Use to re-export only those items that have not yet shipped. In the event of an error in fulfillment, this feature gives you the ability to force the order out again, but only with those items being sent, as the rest have been shipped.
Mark as Accounting Exported Export to an accounting application such as QuickBooks. The export status changes to Accounting Exported.
Mark as Accounting Unexported Changes the export status to Accounting Unexported.
Mark as Sent for Fulfillment Updates the status of the selected orders to Processed--Sent for Fulfillment.
Mark as Shipping Exported Updates the selected orders and marks the orders as Shipping Exported.
Mark as Shipping Unexported Changes the status to Shipping Unexported.
Mark as Not Quick Shipping Changes these orders to show that they will not be Quick Shipped.
Mark as Processed, Shipping Pending Marks the orders as Shipping Pending.
Delete Orders and Return Inventory Deletes orders, and the items are returned to inventory.
Mark as Ordered, Payment Authorized

Use to delete and re-enter an order, yet not re-authorize payment. The basic process is:

  1. Delete the order.
  2. Mark order as Processed Shipping Pending.
  3. Edit the order.
  4. Email customer the paid receipt.
  5. Mark as API Unexported.
  6. Fulfill the order as usual once It moves back up as Ordered, Payment Authorized.
