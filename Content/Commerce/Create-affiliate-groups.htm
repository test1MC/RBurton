Creating affiliate groups

Use this page to manage affiliate groups. Each EOM instance has one default set of fulfillment rules, with priority values set at the supplier level. You can create a unique set of fulfillment rules for each affiliate group that overrides these default rules. Every order has one affiliate, and every affiliate belongs to one affiliate group so each order follows fulfillment rules for their group. There can only be one set of fulfillment rules per affiliate group. More than one affiliate group can be defined on the same EOM instance.

After defining an Affiliate group on the Affiliate Groups page, remember to set up Affiliate emails in Content > Pages > Emails.
Adding an affiliate group
  1. In Commerce > Affiliates > Groups, click the Add New Group button that appears below any other groups already defined.

  2. Provide group information as needed:
    1. Affiliate group Name; required.
    2. Brand name (the Affiliate Group's chain) as used by this group if any. Not the same as 'brand' currently defined on a product.
    3. Language to be used with this group if different from the default; for cases where language is not identified on the order.
    4. Currency if different from the default; for cases where currency is not identified on the order.
    5. Select Disallow Discount Recalculation to disallow discounts if the order originated from this affiliate group. Orders from the affiliate group will not use the EOM promotion engine to recalculate discounts—the discount amount from a 3rd party is maintained.
    6. Select Tax Inclusive Pricing if this group’s product pricing includes VATvalue added tax.

  3. When done, click Save.

 Linking supplier groups to an affiliate group
  1. In Commerce > Affiliates > Groups, select the group to which you want to link one or more supplier groups.

  2. Click Supplier Groups.

  3. To link one or more new supplier groups, click the Link Supplier Group button, and select the groups you want to add.
  4. To remove a supplier group, click its red and white minus icon (-).
  5. To assign group priority, either enter a priority number for a group in the field provided, or hover in it and click the up/down arrows that appear.

  6. To see the list of suppliers and their priorities within that Supplier Group, click the group’s name next to the Group Priority ranking.

    Set up suppliers and priorities in Commerce > Suppliers > Add/Edit.
  7. Click Save when done.
 Defining custom fulfillment rules for an affiliate group

Affiliate group fulfillment rules are used by EOM to choose one or more best suppliers for a specific order. For example, if the rules are set in this order for an affiliate group:

  1. Priority
  2. Rank
  3. Distance
  4. Left Over
  5. Order Volume
  6. SKU Velocity

suppliers within the group will be sorted by:

  • Priority first (note all suppliers with priority = 1 listed first),
  • Rank second (all suppliers with an equal priority will then be sorted by rank),
  • Distance third (all suppliers with equal rank will then be sorted by distance bucket value),
  • Then by left over inventory fourth (all suppliers with the same distance bucket value will then be sorted by left over inventory (which sorts description), and so on.
A group uses the default global fulfillment rules unless you define at least one rule for the group. Note that “Priority” need not be the first rule. See Fulfillment rule descriptions below for basic rule definitions.
To define custom fulfillment rules

  1. In Commerce > Affiliates > Groups, select the group you want to edit.

  2. Click Fulfillment Rules.

  3. To add one or more rules:
    1. Click the Add Rule button.
    2. Select the rules you want to use from the list presented.

    3. Click Add.

  4. Re-order rules in the list by dragging-and dropping.

  5. To remove a rule, click its red and white minus icon (-).
  6. Click Save when finished.
Fulfillment rule descriptions
Rule Description
Distance Takes distance of the supplier from an order into account. All suppliers in the same distance bucket are treated equally.
Left Over Inventory Considers quantity of items remaining in the supplier's inventory after an order is completed.
Order Volume The total volume going to each store.
Priority Takes suppliers’ priority values into account.
Rank Based upon the total number of items in each order. EOM looks at the inventory of each supplier. For example, if an order contains 5 items, all suppliers that can fulfill the entire order will get a value of 5. Suppliers who can partially fulfill the order, will be ranked according to how many items in the order they can ship. Note that the ranking changes for each order depending upon how many items are in it.
SKU Velocity Takes into account the quantity of each SKU a supplier has entered within a given time frame.
Configuring a credit card processor
Defining a credit card processor, or settings for it here, overrides any setting already set up for your website.

  1. In Commerce > Affiliates > Groups, select the group you want to edit.

  2. Click Credit Card.
  3. From the Provider list, select the Credit Card provider to use for this affiliate group.
    Information required differs depending on the provider you select.
  4. Provide information required for the provider selected, and click Save.
 Configuring PayPal for an affiliate group

  1. In Commerce > Affiliates > Groups, select the group you want to define PayPal APIapplication programming interface settings and credentials for.

  2. Click PayPal, and click the Use Affiliate Group Settings toggle to reveal the PayPal settings you can define for this group.

    1. Environment; select if more than one environment in your organization.

    2. API Username; login name for the API; supplied by PayPal.

    3. API Password / Confirm Password; login password for the username supplied.

    4. API Signature; both the username and signature are provided by PayPal.
  3. In the Integration area, choose whether funds are already captured, or will be captured on shipment.
  4. Save.
 Defining a different return address for an affiliate group

For affiliate groups, use this area if the return address and shipping service information for returns needs to be a location that's different than the origin store. This is the address that will be used on return labels if they’re generated and printed.

  1. In Commerce > Affiliates > Groups, select the group you want to define a default return address for.

  2. Click Returns.

  3. Click the Select Return Location button, and search for and select the appropriate returns address for this group.
  4. In the Default Shipping Method for Returns area, select the appropriate Carrier and Service option.
  5. Save.
