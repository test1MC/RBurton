Processing returns

After returns are processed using RMAreturn merchandise authorization requests, they appear on the Returns page. You can also process returns without RMAs and view orders from this page. This keeps a record of the items customers have sent back.

You can view all orders and RMAs on the Returns page.

Allowing for returns on your site

By default, all products on your site are returnable. However, there are several settings you must enable to allow for processing of returns and creation of RMAs. After these settings are enabled, you can customize how you want to process returns.

Warning: Make certain your site is set up to create RMAs and process returns before continuing. Returns and RMAs must be configured properly before they can be used on your site. Refer to knowledge base article #2201, Setting up Your Site to Process RMAs and Returns.

Managing returns

Filter and search for orders

You can filter and search for orders by order number, text, and/or date range. When searching for returns by order numbers that do not have a previous RMA associated with them, the order appears regardless of the date range, making it easy to find the order. You can also search by external order number. This order number is the unique number assigned to the order when the order is created outside of the OMS.

You can also filter and search by:

  • Date Type: Whether it be an RMA date or an order date, you can filter by date type.
  • Return Reason: You can filter by return reasons, if needed. These return reasons (types) are configured at CRM > Communications > Listings. In addition, the Aptos platform allows you to control which return reasons are applicable to online RMA generation. See Managing listings to learn more about return types.
  • Sort by: There are also filters for various sort options in this list menu such as: order number, order date, contact name, order total and salesperson.

You can process returns on this page in one of two ways.

  • Orders without Existing RMAs - When a customer returns merchandise to you without first obtaining an RMA, you can start and complete the return process here.
  • Items with Existing RMA - Items for which an RMA has been created, are displayed in this panel. RMAs are created at Commerce > Orders > Manage.

    • Credit Order: Select if you want to credit the order at this time. If this option is chosen, the Credit Order window opens when this return is saved.

 To process returns with an RMA
  1. In Commerce, select Orders > Returns.
  2. Click Orders With Existing RMAs to expand that area of the screen.
  3. In the search area above this, enter criteria in the fields, and click Search.
  4. Find the RMA you want to process, and from the Select Option list, select the appropriate action.
    1. Cancel Return - Cancels return.
    2. Create Return Printout - Creates printed copy of order notes; opens a pop-up window containing the five most recent order notes. In the Add Condition/New Note field, you can add notes. Click + (Add) to add the notes to the order history. Click Print Now to print a copy of the return history.
    3. Process Received Item - Select this option to process the returned item.
    4. View Order Notes - Opens Order History window; make changes as needed.

    When the item has arrived back at your warehouse, you can process the RMA and complete the return. Select Process Received Item from the list to process the return and close the RMA.

  5. Edit information and settings as needed.

    1. Change Return Reason - The return reason default is the initial setting defined for the RMA. If needed, you can change it.

    2. If the updated return reason does not have an overriding email, the customer will get the default email.

    3. Notify Pending Credit - Notifies the customer in the email of the credit applied, if applicable.

    4. Add to Inventory - Places the item back into your inventory.

    5. Credit Order - Allows for the order to be credited. If checked, upon saving this page, the Credit Order screen opens.

    6. ​Additional Notes to Customer - Place additional order notes here, if needed.

  6. Click Save to process the return.

Crediting orders

After you select items for return, and saved, automatically looks for three fields for each returned item, and opens and pre-populate the Credit Order window with those amounts. These fields are:

  • returnable credit amount (located in the credit column)
  • Shipping Credit
  • Tax Credit

These fields are populated via the 'ConfirmShippedOrder' API method located at: https://api.Aptos.com/wiki/Order ConfirmShippedOrder.aspx.

After processing the credit for the return, places the "return reason" selected in the Reason for Credit field and ignores the value from the "Credit Reason" selection if your site uses the GetOrderCredits API call to get the actual return reason when a credit is applied.

If the items being returned have not had a returnable credit amount specified, a message is displayed at the top of the Credit Order window stating,

The items being returned have not had a returnable credit amount specified and the credit cannot be applied automatically. Please manually apply a detailed level credit/refund.

This means that the credit cannot be applied automatically because the 'returnable credit field' has not been populated. This happens for any orders that do not have this field populated or if your site is not using the API. When this is the case, you will need to go to Commerce > Orders > View > Options > Credit Order and manually apply a detailed credit/refund.

The Credit Order page defaults to the 'Detailed Credit' view and only shows those items that are being returned. This helps to reduce user error when an order is credited. See Using the View Order window to learn more about crediting orders .
